## 一. 文章内容概括

#### 解决了什么问题？

- **推理延迟高**：现有的世界动作模型 (WAM) 大多遵循“先想象后执行”的范式，它们需要先生成未来画面，然后再基于此预测动作，由于需要对视频进行迭代去噪，这导致了测试时极高的推理延迟 。  
- **未来想象的必要性存疑**：目前尚不清楚 WAM 在测试阶段显式生成未来画面是否真的是实现强大动作性能所必需的，还是其优势主要来源于训练阶段的视频建模。  
- **核心因素难以解耦**：现有的 WAM 系统通常将训练时的视频预测目标与推理时的显式未来生成这两个因素纠缠在一起，难以确定究竟是哪一个因素带来了性能提升。  

#### 怎么解决的？

- **解耦训练与推理**：提出了 Fast-WAM，这是一种将训练时的视频协同训练与推理时的显式未来生成完全解耦的模型架构 。它在训练阶段保留了视频协同训练，但在测试阶段跳过了未来画面的预测 。  
- **单次前向传播编码**：不使用预训练视频模型在推理时迭代合成未来帧，而是将预训练的视频扩散 Transformer (DiT) 重新用作单次前向传播的世界编码器，直接提取潜在世界表示用于动作生成 。  
- **受控变体对比**：构建了多个受控变体（如遵循联合生成的 Fast-WAM-Joint、遵循视频后动作的 Fast-WAM-IDM，以及无视频协同训练变体），通过对比实验证明，WAM 的优势主要来自于视频协同训练目标本身，而推理时的未来生成并非关键 。  
- **实现实时高效控制**：Fast-WAM 在不依赖具身预训练的情况下，在仿真和真实世界任务中取得了与最先进方法相当的结果，并且运行延迟仅为 190 毫秒，比现有的“先想象后执行”的 WAM 快 4 倍以上 。

### 二. 模型结构

Fast-WAM 采用具有共享注意力的混合 Transformer (Mixture-of-Transformer, MoT) 架构，由一个视频 DiT 和一个动作专家 DiT 组成。  

![model](/Users/oucunxu/study/实习面试/八股/具身经典工作/WAM/Fast-WAM/images/model.png)

**A. 整体架构**

- **主干网络 (Backbone)**：建立在 Wan2.2-5B 预训练视频扩散 Transformer (DiT) 的基础上，作为世界建模的骨干网络 。同时复用了其预训练的 T5 文本编码器和视频 VAE 。  
- **动作专家 (Action Expert)**：在主干之上引入了一个动作专家 DiT 用于生成动作块 (action chunk) 。其架构与视频分支相同，但隐藏层维度减小到 `d_a=1024`，参数量约为 1B，模型总参数量为 6B 。  

**B. 输入编码与注意力掩码**

- **输入分组**：输入 token 被组织为三组：第一帧观测的干净潜在 token（作为共享的视觉锚点）；未来视频帧的噪声潜在 token（仅在训练期间用于视频建模）；以及用于生成动作的动作 token 。  
- **交叉注意力**：所有分组的 token 都通过交叉注意力 (cross-attention) 机制关注由语言指令提取的特征向量 。  
- **结构化注意力掩码**：模型通过一个特殊的注意力掩码来控制信息流 。在训练期间，未来视频的噪声 token 在视频分支内进行双向注意力计算，并能访问干净的首帧 token ；动作 token 在动作分支内双向计算，也能访问干净的首帧 token 。关键在于，**动作 token 和未来的视频 token 互不关注，而干净的首帧 token 也不关注任何其他 token**，这确保了两者共享相同的视觉上下文，同时防止未来信息泄漏到动作分支 。  

**C. 推理时的极简架构**

- 在推理时，Fast-WAM 完全移除了未来视频分支 。  
- 模型只保留干净的首帧潜在 token，通过视频骨干网络进行单次前向传播，为动作专家生成潜在的世界特征 。由于没有实例化未来的噪声视频 token，也无需进行显式的视频去噪，因此大幅降低了推理成本 。

### 三. 训练与推理流程

**A. 训练流程 (Joint Flow Matching Objective)**

Fast-WAM 使用针对动作 token 和未来视频潜变量的联合流匹配 (Flow Matching) 目标进行训练 。  

1. **时间步采样与加噪**：给定目标变量 `y` （动作块 `a_1:H` 或未来视频潜变量 `z_1:T`），采样高斯噪声 `ε~ N(0,I)` 和时间步 `t∈(0,1)`，构建插值样本：
   ```text
   y_t=(1-t)y+t~ε
   ```
   其中时间步 `t` 遵循对数正态分布噪声调度 。  

2. **视频协同训练 (Video Co-training)**：通过视频预测目标，促使视频 DiT 学习编码具有物理意义的运动和交互结构 。针对预训练 VAE 提取的未来视频帧潜在 token `z_1:T` 优化速度场预测：  
   ```text
   L_vid=L_FM(z_1:T)
   ```

3. **动作预测学习**：针对动作序列优化标准流匹配目标：
   ```text
   L_act=L_FM(a_1:H)
   ```

4. **计算整体损失**：联合训练的总体损失函数平衡了动作学习和视频协同训练：
   ```text
   L=L_act+λL_vid
   ```
   其中 `λ` 用于平衡这两个任务的权重 。  

**B. 闭环推理流程**

不同于“先想象后执行”的框架，Fast-WAM 在测试时类似于一个直接策略模型，但在表示层面上依然保留了基于 WAM 风格视频建模的基础。这彻底避免了视频迭代去噪，使实时闭环控制成为可能 。

1. **单次前向传播**：Video DiT 首先处理首帧干净 token (`f_0`)，并通过交叉注意力关注指令 `l`，执行一次前向传播。
2. **KV Cache 缓存**：在这一次传播中产生的关于 `f_0` 的 Key 和 Value 矩阵会被缓存起来（如图 1(C) ） 。
3. **动作去噪迭代**：随后，只有噪声动作 token 输入到 Action DiT 中 。在 Action DiT 进行迭代去噪（10 步）计算时，动作 token 会根据注意力掩码，关注刚才缓存好的 `f_0` 的 K/V 特征 。  

> Video DiT 可扮演了两种角色：
>
> - **去噪器/世界模型**：训练时，它被用来对未来视频帧进行去噪 。模型通过联合流匹配目标，迫使 Video DiT 学习预测未来视频画面的演化（即视频协同训练 `L_vid`） 。这个过程促使它的大脑里学到了物理世界中物体的运动和交互规律 。  
> - **单步特征编码器**：由于它在训练时已经被逼着学懂了物理规律，推理时它作为一个强大的单步视觉编码器，把当前的观测 `o` 一口吞下，吐出一个富含物理先验的“潜在表示”，直接辅助动作预测 。
