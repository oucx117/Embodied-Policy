## FAST: EFFICIENT ACTION TOKENIZATION FOR VISION-LANGUAGE-ACTION MODELS

### 一. 工作动机

**核心问题**：当前主流的单体式 VLA 模型（如 RT-2, OpenVLA）所采用的“朴素”动作 tokenization 方法（即逐时间步、逐维度地分箱 bins），在处理高频、灵巧的机器人操作任务时表现不佳。首先，高频数据意味着一个动作片段（action chunk）会包含大量时间步，而**朴素方法会为每个时间步都生成多个 token，导致序列过长，推理时间过长**；其次，在高频控制下，相邻动作高度相似，经过粗糙的离散化分箱后，它们很大概率会落入同一个区间，从而被映射为完全相同的 token，导致**生成的 token 序列信息含大量冗余，学习信号微弱**，模型很容易陷入“简单复制上一个动作”的局部最优解中，而无法学会真正的复杂技能。

**核心思想**：将机器人动作块（action chunk）视为一个连续的时序信号，并借鉴信号处理领域的思想，先对其进行**离散余弦变换**（DCT）转化为频域，使得**重要的低频信息和冗余的高频信息得以分离，有效地将核心信息集中在少数几个系数上**，之后通过**量化压缩**去除高频冗余信息，并借助**字节对编码**（BPE）进行进一步压缩，最终得到短小但信息密度高的 token 序列，从而极大地提升自回归 VLA 模型的学习效率和性能 。

---

### 二. FAST 方法

FAST 是一种基于压缩的机器人动作序列 tokenization 流程，主要由五个串行步骤构成：

1. **动作归一化**
   - **输入**：一个包含多个时间步的连续机器人动作块。
   - **流程**：为了消除不同维度（如位置和角度）间的**量纲差异**并增强对异常值的鲁棒性，该步骤将训练数据集中每个动作维度的第 1 和第 99 百分位数的值线性映射到 $[-1, 1]$ 的区间内 。
2. **离散余弦变换（DCT）**
   - **输入**：归一化后的动作块。
   - **流程**：对每个动作维度**独立地**应用 DCT，将时域信号转换到频域 。这一步具备“能量集中”的特性，即将描述动作整体形状的**核心信息**集中到少数几个**低频系数**上，而将描述微小抖动的次要信息分布到大量**高频系数**上 。
3. **量化与压缩**
   - **输入**：DCT 变换后的频域系数矩阵。
   - **流程**：这是一个**有损压缩**步骤。通过一个“缩放并四舍五入”的操作，将浮点数系数转换为整数，并使得那些值较小的高频系数变为 0，从而形成一个稀疏的整数矩阵，实现**对次要信息的丢弃和对数据的大幅压缩** 。
4. **展平**
   - **输入**：稀疏的整数系数矩阵。
   - **流程**：将二维的系数矩阵转换为一维的整数序列。采用“列优先”的顺序，即**先拼接所有维度的最低频系数，再拼接所有维度的次低频系数**，以此类推。这样做是为了**让模型在自回归预测时优先生成决定动作整体轮廓的低频信息，以增强策略的稳定性**。
5. **字节对编码（BPE）**
   - **输入**：展平后的一维稀疏整数序列。
   - **流程**：这是一个**无损压缩**步骤。通过训练一个 BPE tokenizer，将序列中频繁出现的整数组合（尤其是连续的 0）合并成一个新的、单一的 token 。
   - **输出**：最终的、短小且稠密的 action token 序列，该序列拥有一个固定大小的词汇表，可以方便地集成到 VLA 模型中 。

- **FAST+**
  - **定义**：是一个基于 FAST 流程，在一个包含约 100 万条真实机器人动作轨迹的大规模、跨平台混合数据集上**预训练好**的通用动作 tokenizer；
  - **核心特点**：可以作为一个即插即用的**“黑盒”工具**，直接对来自各种新机器人（包括移动机器人、灵巧手、人形机器人等）的动作序列进行高效压缩，无需重新训练，极大地提升了便捷性和通用性 。实验证明其性能与为特定数据集定制的 FAST tokenizer 高度接近 。

### 三. 实验

- **Q1: 在高频、灵巧任务上，FAST 是否显著优于传统方法？**
  - **实验设置**：在一系列真实机器人任务上，对比了 Naïve (传统分箱方法)、FSQ (另一种压缩方法) 以及 FAST/FAST+ 的性能，任务频率从 15Hz 到 50Hz 不等 。
  - **实验结论**：Naïve 方法在 20Hz 的 Table Bussing 和 50Hz 的 T-Shirt Folding 任务上完全失败，无法取得任何进展，而基于压缩的 FAST 方法则在所有任务上都取得了优异的性能，证明了其在处理高频数据上的绝对优势。
- **Q2：FAST 能否让自回归 VLA 的性能和训练效率媲美先进的 Diffusion VLA？**
  - **实验设置**：将集成了 FAST 的自回归 π0 模型 (π0-FAST) 与原始的、基于扩散模型的 π0 VLA 在多个任务上进行性能和训练速度的对比 。
  - **实验结论**：π0-FAST 在最终性能上成功匹配了顶尖的 π0 扩散模型 。更重要的是，其**收敛速度显著更快**，达到同等性能所需的**训练计算量（GPU 小时数）最多可减少 5 倍** 。
- **Q3：通用 tokenizer FAST+ 是否能有效泛化到未见过的机器人平台？**
  - **实验设置**：在一系列**完全没有**被用于训练 FAST+ 的新数据集上测试其压缩性能和策略训练效果，这些数据集涵盖灵巧手、人形机器人和自动驾驶汽车等 。
  - **实验结论**：FAST+ 在这些未见过的平台上展现了良好的压缩能力。在策略训练中，使用通用的 FAST+ tokenizer 的性能与使用为特定任务定制的 FAST tokenizer 的性能几乎没有差别，证明了其强大的泛化能力和作为“默认”选项的有效性 。

---

### 四、总结

- **主要贡献**：提出了 FAST，一种高效的、基于压缩的动作 tokenization 方案。它首次成功地让自回归 VLA 模型能够学习高频、灵巧的机器人操作任务，并显著提升了大规模 VLA 模型的训练效率 。

- **三大优势总结**：

  1. **攻克高频任务**：通过频域压缩解决了传统 tokenization 方法在高频数据上的信息冗余和学习信号弱的问题 。
  2. **超高训练效率**：使自回归 VLA 能够以远低于扩散模型的计算成本（最多快 5 倍），达到与之相媲美的性能 。
  3. **便捷与通用**：提供了预训练的 FAST+ 工具，可作为“即插即用”的黑盒 tokenizer，方便地应用于各种机器人平台 。

- **局限性**：

  1. **推理速度**：尽管训练快，但自回归的 π0-FAST 在推理时需要逐个 token 生成，速度慢于并行生成整个动作块的 π0 扩散模型 。

- **未来工作**：

  - **动作 Tokenizer 的扩展与探索** 

    * 扩展应用平台：将 FAST 方法的实际策略性能测试从静态机械臂扩展到更多类型的机器人上，例如移动机器人、灵巧手和人形机器人；
    * 探索新压缩方案：研究除了 FAST 所用的 DCT 之外的其他压缩技术；

    * 结合不同解码方法：测试将基于压缩的动作编码（如 FAST）与非自回归的解码方法（如扩散模型）相结合的效果。

  - **VLA 架构的深入比较**

    * 目前关于自回归 VLA 和扩散 VLA 哪种架构更好尚无定论。未来的工作需要更仔细地探索和比较这两种架构在训练速度、语言理解能力和模型表达能力等方面的优缺点。

  - **提升推理速度** 

    * 尽管 π0-FAST 模型性能优越，但其推理速度慢于扩散模型版本的 π0。为了让模型能够胜任高度动态的任务，未来的工作需要探索提升自回归 VLA 模型推理速度的方法。这可以借鉴大型语言模型领域已有的大量推理优化技术 。
