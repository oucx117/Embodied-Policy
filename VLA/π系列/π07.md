## `\pi_{0.7}`: A Steerable Generalist Robotic Foundation Model with Emergent Capabilities

### 一. 工作动机

**核心问题**：虽然 VLA 在规模和能力上取得了显著进步，但它们往往缺乏**“组合泛化”**的能力，即难以像大语言模型那样**将训练数据中的不同能力重新组合来解决全新的任务**。此外，为了获取广泛的经验，如果天真地将大量来源多样、质量参差不齐的数据（包括自主运行产生的次优数据或失败案例）混合训练，会导致模型平均化不同策略而使性能大打折扣。

**核心思想**：`\pi_{0.7}` 提出在训练时引入**“多样化上下文条件”**或**“多模态提示词”**。这种详细的 Prompt 不仅告诉模型“要做什么（What）”，还包含**关于策略和任务表现的元数据以及子目标图像**，告诉模型“怎么做（How）”。这种设计解决了混合次优数据带来的模糊性，使得模型不仅能吸收庞杂且低质量的数据而不受损，反而能够从中学习并展现出强大的开箱即用能力、跨实体泛化能力和零样本组合泛化能力。

### 二. `\pi_{0.7}` 模型

`\pi_{0.7}` 是一个 **5B 参数**的通用机器人基础模型，在 `\pi_{0.6}` 和 MEM（多尺度具身记忆）架构的基础上进行了扩展，核心是引入了极其丰富的多模态上下文条件。

![pi07_model](./images/pi07_model.png)

**A. 整体架构**

类似于 `\pi_{0}` 的结构，但整合了更强大的组件：

- **VLM 主干**：基于 **Gemma3 4B** 参数视觉-语言模型（VLM）初始化。
- **视觉历史编码器**：采用了 MEM 风格的 **400M 参数 SigLIP** 视觉编码器，对历史观察帧进行时间和空间压缩。
- **动作专家**：**860M 参数**的**流匹配**（Flow Matching）Transformer，专用于预测。

**B. 多模态提示词与上下文 (`C_t`)**

为了处理异构数据集，`\pi_{0.7}` 的输入 Prompt 包含了前所未有的丰富信息：

- **子任务指令**：除了高层次的“清理厨房”外，还包含类似“打开冰箱门”的细粒度文本，便于进行分步指导。
- **子目标图像**：由一个独立的轻量级世界模型（基于 BAGEL 14B）生成的**多视角未来状态图像**，展示**当前子步骤结束时应该呈现的样子**，为动作提供了更丰富的空间和视觉约束。
- **回合元数据**：模型在训练时必须理解数据的质量。为此引入了关于当前轨迹的三个维度标签：
  - **整体速度 (Speed)**：以步数离散化表示任务耗时（episode-level，针对高层次）。
  - **整体质量 (Quality) **：1 到 5 的主观评分（episode-level，针对高层次）。
  - **失误 (Mistake)**：布尔值标签，标记片段中是否有抓取失败等失误（segment-level，针对子步骤）。
- **控制模式**：用文本指定是进行关节级还是末端执行器控制。

> **为什么要在 Prompt 中加入“元数据”？**
>
> 传统的模型只能从人类高分示范中学习，但 `\pi_{0.7}` 会把模型过去失败的、或者在评估时收集的带失误的自主轨迹也扔进训练集，通过贴上 Quality=2 或 Mistake=True 的标签，模型就能明白“这是一个糟糕的演示，我应该知道其对应什么动作，但在测试时我不会这么选”。

**C. 注意力掩码策略：块状因果掩码**

1. **观察帧块**：包含当前及历史的摄像机图像和机器人的本体感知状态。

   * **块内注意力**：**双向注意力**。同一时间步或不同视角的观察帧特征在这一阶段进行充分的全局融合。

   - **对外部的可见性**：**仅对自己可见**。由于处于序列的最前端，基于因果掩码的特性，它绝对看不到未来的文本、子目标和动作信息。

2. **文本块**：包含任务指令、回合元数据（如速度、质量）以及控制模式等提示词。

   - **块内注意力**：**因果注意力**。文本 Token 像标准的语言模型一样，只能看到自身以及排在它前面的文本 Token。

   - **对外部的可见性**：**向前看**。它可以看到排在它前面的“观察帧块”信息。

3. **子目标图像块**：由世界模型生成的未来期望状态图像。

   - **块内注意力**：**双向注意力**。子目标图像的各个 Token 之间需要全局交互以理解完整的未来视觉场景。

   - **对外部的可见性**：**向前看**。作为一个附加的块状因果双向块，它被放置在文本提示词之后。因此，子目标图像块可以关注前面的“观察帧块” 和“文本块”信息。

4. **离散动作块 (FAST Tokens)**：这是 VLM 主干在训练阶段用来计算离散交叉熵损失的词元。

   - **块内注意力**：**因果注意力**。由于是要预测离散的下一个动作 Token，必须严格自回归。

   - **对外部的可见性**：可以看到前面的多模态上下文（观察帧、文本、子目标图像），但**绝对看不到流匹配连续动作块**。

5. **流匹配连续动作块 (Flow Actions)**：这是动作专家（Action Expert）用来生成 50 步连续动作轨迹的模块。

   - **块内注意力**：**双向注意力**。这 50 个连续动作 Token 互相之间是完全可见的，因为它们需要同时被去噪并输出一个平滑的动作块。

   - **对外部的可见性**：它们可以关注 VLM 主干的所有激活状态（即前面的观察帧、文本、子目标图像）。但是，**它们绝对看不到离散动作块 (FAST Tokens)**。

### 三. 训练与推理

- **训练流程（模仿学习）**：

  1. **构建海量异构数据**：不仅仅使用**高质量示范**，还大量使用了模型（如 `\pi_{0.6}`）在之前测试中产生的**自主评估数据**、**人为干预数据**、甚至**失败案例**，还补充了**人类第一人称视频**和**互联网非机器人数据**。
  2. **带噪目标采样**：处理子目标图像时，**混合使用未来真实帧（0-4秒后）与世界模型生成的图像**，以减弱训练和测试时真实/生成图像之间的分布差异。
  3. **Prompt Dropout**：训练时会**以不同概率随机丢弃 Prompt 的各部分**（例如，只有 25% 的样本带有子目标图像，有 15% 的数据完全丢弃元数据），使得模型在测试时可以灵活接受任意组合的提示。
  4. **模型更新**：保持 VLM 的离散交叉熵训练，动作专家通过匹配连续动作分布（Flow Matching）进行优化。

- **推理流程**：

  1. **元数据设定**：在测试时**“向模型许愿”**，直接 Prompt 输入：Quality=5、Mistake=false 以及任务历史的第 15 百分位的高速度。

  2. **异步生成**：子任务指令和世界模型生成的子目标图像由于计算开销大，会在独立的后台线程中**异步刷新**，VLA 推理始终读取最新的可用生成结果。

     > `\pi_{0.7}` 的核心动作输出频率很高（比如 50Hz），但它的“世界模型”（基于 14B 参数的 BAGEL）生成一张子目标图像是非常耗时的。  如果在每次动作前都等世界模型生成完图片，机器人就会严重卡顿。因此，系统采用了异步推理：视觉子目标和子任务指令的生成在单独的线程中进行，而前台的 VLA 推理则始终读取最新的可用结果。刷新规则：当语义意图发生改变（即产生了新的子任务指令 ），或者距离上一次生成子目标图像已经过了 4 秒，后台就会触发一次新的刷新。

  3. **去噪与引导**：通过 **5 步去噪迭代**生成 **50 步的动作块**，然后仅执行**前 15 或 25 步**。生成过程中可对回合元数据使用**无分类器引导**，将预测进一步推向高速度或高分的元数据方向。

     > **什么是无分类器引导？**
     >
     > 在推理阶段生成动作时，模型不仅要在带有条件  （比如 `Quality=5`）的情况下进行一次去噪预测 ，同时还要在隐去该条件（也就是无条件模式 ）的情况下再进行一次预测 。通过计算两者之间的差异向量，并乘以一个放缩系数 （大于 1），得到最终的动作更新（去噪）方向：
     > ```text
     > \nabla_{a}\log\pi_{\theta}(a_{t:t+H}|o_{t},\mathcal{C}_{t}) + \beta\left(\nabla_{a}\log\pi_{\theta}(a_{t:t+H}|o_{t},\mathcal{C}_{t}) - \nabla_{a}\log\pi_{\theta}(a_{t:t+H}|o_{t},\mathcal{C}_{t}^{uncond})\right)
     > ```
     > 相当于告诉模型：“你不仅要生成高质量的动作，你还要给我极力远离你那个平庸的无条件默认习惯！”

### 四. 实验

- **研究问题一：`\pi_{0.7}` 的“开箱即用”灵巧操作能力能否匹敌专长模型？**
  - **实验设置**：对比唯一的 `\pi_{0.7}` 通用模型与通过强化学习（RL）或 SFT 针对单一任务专门微调的 `\pi_{0.6}` 专家模型，测试煮浓缩咖啡、叠衣服、组装纸箱等高难度任务。
  - **实验结论**：无需任何任务特定的后训练，`\pi_{0.7}` 就匹配甚至超越了 RL 专家模型（例如在叠衣和组装纸箱上吞吐量更高）。消融实验证明，这完全归功于“保留评估产生的次优数据”以及“加入元数据 Prompt”。
- **研究问题二：`\pi_{0.7}` 对复杂语言指令的遵循能力如何？**
  - **实验设置**：在 4 个没见过的厨房和 2 个没见过的卧室中，用非标准、罕见的指代语言（如“拿起用来喝汤的物品”或逆向思维指令如“把碗碟扔进垃圾桶”）测试模型。
  - **实验结论**：`\pi_{0.7}` 能够有效克服预训练数据中的固有偏见，遵循逆向指令。特别是当接入子目标生成（世界模型）后，语言遵循能力进一步跃升，大幅超越了以前的模型。
- **研究问题三：`\pi_{0.7}` 能否实现零样本的跨硬件实体泛化？**
  - **实验设置**：将使用小型双臂机器人收集的“叠衬衫/毛巾”数据，直接零样本迁移到体型大得多、动力学完全不同的 UR5e 工业双臂系统上运行，并与专业人类遥操作员的初次盲测表现对比。
  - **实验结论**：`\pi_{0.7}` 取得了与人类专家相当的成功率（80% vs. 80.6%）。最令人惊讶的是，模型并非在目标机器人上生搬硬套源机器人的动作轨迹，而是**涌现出了自适应的策略**（例如由于手臂更长，它把双臂协同改为了单臂抓放策略）。
- **研究问题四：`\pi_{0.7}` 能否像组合积木一样解决未见过的全新任务？**
  - **实验设置**：测试完全没有采集过低级动作数据的全新短视野任务（如擦耳机、转风扇）和长视野任务（如使用空气炸锅烤红薯）。
  - **实验结论**：由于其极强的泛化性和听话能力，人类可以通过给出“打开门 -> 抓红薯 -> 放入炸锅”的**语言指导（Coaching）**让机器人完成全新的复杂长视野任务。随后，这些纯文本的对话数据可以提炼成一个高层自主策略，实现完全无需额外遥操作数据的全新任务自主运行。
- **扩展研究：数据集多样性是否等于性能？**
  - **实验结论**：如果没有元数据介入，盲目增加混杂质量的数据会导致模型变弱；但加入元数据后，`\pi_{0.7}` 能够随着数据量（哪怕是劣质数据）的增加而持续变强。同时，正是数据集中高度的任务多样性直接驱动了模型在全新任务上的组合泛化能力。

### 五. 局限性

- **零样本成功率的天花板**：虽然泛化范围广，但 `\pi_{0.7}` 在完全未见过的任务或硬件组合上的零样本成功率在 60-80% 之间，依然低于其在分布内任务上 90% 以上的成功率，要达到工业级可靠性仍需通过提示或少量 RL 微调进行优化。
- **“未见过（Unseen）”的定义变得模糊**：由于预训练混合了规模如此庞大且包含互联网图像、视频的海量异构数据，这使得研究人员越来越难界定一个任务是真正的“全新（Novel）”，还是模型在其他场景中见过相似片段后的“重组（Remixing）”。
