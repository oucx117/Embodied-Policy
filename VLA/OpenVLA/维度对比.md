**数学符号**：

- `C`: 相机视角数量
- `P`: 单张图片的 patch 数
- `L`: 语言指令的 Token 数量
- `D_a`: 机器人的动作自由度
- `K`: 动作分块大小
- `T_o`: 观测历史的步数
- `D_emb`: 模型主干的隐藏层维度

------

### 1. 核心维度对比表

| **特性 / 模型**           | OpenVLA                      | OpenVLA-OFT                                         | ACT                                                       | Diffusion Policy                    |
| ------------------------- | ---------------------------- | --------------------------------------------------- | --------------------------------------------------------- | ----------------------------------- |
| **动作生成范式**          | 离散 Token，串行生成         | 连续数值，并行生成                                  | 连续数值，并行生成                                        | 连续数值，多次循环去噪生成          |
| **动作 Token 颗粒度**     | 1 Token = 1 个动作自由度     | 1 Token = 1 个动作自由度                            | 1 Token = 1 个完整时间步动作                              | 1 Token = 1 个完整时间步动作        |
| **输入序列长度 (Tokens)** | `C × P + L (每步递增)`  | `C × 256 + 1 (状态) + L + K × D_a` | 编码器: `C × P + 2(状态+z)` ，解码器: `K (位置查询)` | Cross-Attn：`T_o`，Self-Attn: `K+1` |
| **网络直接输出维度**      | `(1, Vocab_Size)`    | `(K × D_a, 1)`                                 | `(K, D_a)`                                                | `(K, D_a)`                          |
| **最终物理动作维度**      | `(1, D_a)` (需循环 `D_a` 次) | `(K, D_a)`                                          | `(K, D_a)`                                                | `(K, D_a)`                          |

### 2. 详细计算流拆解

#### ① 原版 OpenVLA：纯粹的“语言接龙”

- **输入端：** 把图像 Patch 和语言 Token 拼在一起送进去。在生成动作时，把上一步预测的动作 Token 拼在末尾，一次只能前进一步。
  - 序列形态：`[图像] + [文本] + [动作1_Token] + [动作2_Token]...`
- **输出端：** 经过 Softmax，输出词汇表（比如 32000 维）的概率分布，选出概率最大的词。
- **痛点：** 要得到 `K` 步动作，需要串行执行 `K × D_a` 次前向传播，速度极慢。

#### ② OpenVLA-OFT：用“填空题”实现并行

- **输入端：** 在序列末尾强行塞入 `K × D_a` 个**只带有位置编码的空占位符**。这里的颗粒度极细，**1 个自由度占据 1 个 Token 的位置**。
  - 序列形态：`[图像] + [状态] + [文本] + [占位符1] ... + [占位符 K*Da]`
- **输出端：** 这 `K × D_a` 个占位符经过双向注意力吸收信息后，各自通过 MLP（回归头），被独立压缩成 `K × D_a` 个**连续标量（1D数值）**。然后在逻辑上 Resize 成 `(K, D_a)` 的矩阵。
- **亮点：** 继承了 LLM 强大的长序列细粒度注意力机制，但把分类变成了直接回归，实现了 1 次前向传播生成所有动作。

#### ③ ACT：整体论的“动作包裹”

- **输入端（解码器）：** ACT 不像大语言模型那样把所有东西拼成一条长序列。它的解码器输入仅仅是 `K` 个固定的位置查询（Queries）。**1 个 Query 对应未来 1 个时间步的所有关节。**
  - 序列形态：`[Query_步1], [Query_步2] ... [Query_步K]`
- **输出端：** 解码器吐出 `K` 个 `D_emb` 维的高维向量。随后，每个向量被 MLP 直接映射成 14 维的关节角度 `(K, 14)`。
- **亮点：** Transformer 内部只处理时间维度的注意力交互，物理自由度在特征层面被“打包”了，计算量小，生成极其平滑。

#### ④ Diffusion Policy (Transformer版)：条件的极致压缩

- **输入端（观测条件）：** 使用空间 Softmax，将 `C` 张图片和状态极致压缩为 1 个 Token。看过去 `T_o` 帧，就只有 `T_o` 个 Token 作为条件。
  - 条件形态：`[过去帧_Token], [当前帧_Token]`
- **输入端（去噪目标）：** 把 `K` 步的含噪动作当作 `K` 个 Token 送入自注意力层。同样是 **1 个 Token 代表 1 个完整的时间步**。
  - 序列形态：`[步数k_Token] + [含噪动作_步1] ... + [含噪动作_步K]`
- **输出端：** 输出 `K` 个预测噪声，维度是 `(K, D_a)`，用来从输入中减去噪声。
- **亮点：** 极短的 Token 序列保证了即便需要循环计算 `K` 次（去噪），推理速度依然能满足真实机器人的高频控制（10Hz~25Hz）。
