## OpenVLA: An Open-Source Vision-Language-Action Model

### 一. 工作动机

**核心问题**：尽管像 RT-2-X 这样的视觉-语言-动作（VLA）模型展现了强大的通用能力，但它们大多是**闭源的**，模型细节、训练代码和权重均不公开，这极大地阻碍了更广泛的学术研究和社区发展 。此外，如何**高效地**将这些巨大的模型**微调**到新任务、新机器人上，尤其是在**消费级硬件**上进行微调，也是一个未被充分探索的关键问题 。

**核心思想**：创建一个强大的、**完全开源的**、且在参数效率上超越现有闭源模型的 VLA，旨在提供一个高性能的通用机器人操纵基础模型，让整个社区都能方便地在其之上进行研究、微调和应用。

---

### 二. OpenVLA 模型

OpenVLA 并非一个全新的架构，而是通过在一个强大的、开源的预训练视觉语言模型（VLM）基础上，用大规模机器人数据进行微调，从而得到的 VLA 模型 。

![model](./images/model.png)

* **输入**：

  - **图像**：机器人摄像头的**单帧**实时观测图像，分辨率为 224 $\times$ 224 。

  - **指令**：自然语言指令，如 "Wipe the table" 。
* **模型骨干 (Backbone)**：

  - **基础VLM**：基于一个开源的、强大的VLM模型 **Prismatic-7B** 。

  - **语言模型核心**：**Llama 2 7B**，一个拥有70亿参数的强大语言模型，作为决策中枢 。

  - **视觉编码器 (关键创新)**：采用**融合视觉编码器**，将两个强大的预训练视觉模型的特征沿通道进行拼接 ，以提供更全面的视觉信息，含有 600M 参数：
    - **SigLIP**：提供与语言对齐的**语义特征**（粗粒度信息，回答“是什么”） 。
    - **DINOv2**：提供精细的**空间/几何特征**（细粒度信息，回答“在哪里/怎么样”） 。

  - **投影器**：一个简单的 2 层 MLP，负责将视觉编码器输出的特征向量“翻译”到 Llama 2 能够理解的文本嵌入空间中 。
* **动作输出**：

  - **核心方法**：与 RT-2 类似，将机器人动作视为一种特殊的“语言”，让LLM直接生成动作词元（token） 。

  - **离散化**：每个连续的动作维度被分别划分为 **256 个“桶”(bins)** 。

  - **离散化范围 (关键改进)**：RT-2是将 [min, max] 均分为 256 个区间，而 OpenVLA 则是根据**第1至第99百分位数**来划分区间，从而排除了异常值的影响，使得绝大多数有效动作发生的区域的 bin 个数不会受异常值影响而减少，提升了动作的有效精度。

  - **Tokenization**：将代表动作值的256个整数（0-255）映射到 Llama 词汇表中 **256个最不常用的词元**上。

  - **训练目标**：使用标准的**交叉熵损失函数**进行下一词元预测。
* **训练/微调细节**：

  - **大规模微调**：将一个通用的、预训练好的 VLM 转化成一个专业的机器人策略模型 OpenVLA。这是在一个大型 GPU 集群上，对 Prismatic-7B 进行的**全参数微调**来实现的。这个过程计算量巨大，作者也称之为 VLA training，并发现与典型的VLM训练（1-2个epoch）不同，该训练需要**极多的训练轮次**才能达到最佳性能，最终模型在数据集上完整训练了 **27个轮次** 。
  - **小规模微调**：将作者们已经训练好的、公开发布的 OpenVLA 模型，适配到用户自己的新任务或新机器人上。
  - **微调视觉编码器**：与 VLM 微调通常冻结视觉编码器不同，作者发现 **微调视觉编码器** 对于获得好的VLA性能至关重要（无论是大规模微调还是小规模微调），因为这有助于模型捕捉机器人操控所需的细粒度空间信息 。

### 三. 实验

- **数据集**：

  - 基于 **Open X-Embodiment (OpenX)** 数据集，包含 **97万条**真实世界机器人演示轨迹；
  - **数据筛选策略**：
    1. **确保输入和输出在不同的数据来源中遵循统一的格式**：限制数据满足以下三个条件：
       1. 操纵类任务；
       2. 包含至少一个第三人称视角摄像头（保证了视觉输入的一致性）；
       3. 必须是单臂末端执行器控制（意味着动作包含7个自由度，保证了动作输出的一致性）
    2. **确保最终的训练混合数据中，机器人形态、任务和场景的配比均衡**：借鉴 Octo 模型的经验，使用了该论文的数据混合权重，即提高那些任务和场景多样性更丰富的数据集的权重，降低多样性较差的数据集的权重或直接将其移除。

- **研究问题一：OpenVLA 与先前的通用机器人策略模型的性能对比如何？**

  - **对比基线模型 (Baselines)**：
    - `RT-1-X`：一个从零开始在 OpenX 数据集子集上训练的 35M 参数 Transformer 模型 。
    - `Octo`：当时**开源模型中的 SOTA**，一个 93M 参数的通用模型，同样在 OpenX 数据集子集上从零开始训练 。
    - `RT-2-X`：当时**最强的、闭源的** VLA 模型，拥有 55B 参数，利用了互联网规模的视觉和语言预训练 。

  * **实验设置**：

    - 对模型进行“**开箱即用**”式的直接评估，不经过任何额外微调 。

    - 在两个不同的机器人平台（BridgeData V2 中的 WidowX 机器人和 RT-1/RT-2 中使用的谷歌机器人）上进行测试 。

    - 评估涵盖了多种泛化能力维度，包括视觉泛化（新背景、干扰物）、运动泛化（新位置、姿态）、物理泛化（新尺寸、形状）和语义泛化（新物体、新指令）。

  * **实验结果与结论**：

    - **OpenVLA 性能全面领先**：在所有通用模型中，OpenVLA 的综合表现最强 。
    - **预训练优势显著**：相比从零开始训练的 RT-1-X 和 Octo，基于 VLM 的 RT-2-X 和 OpenVLA 表现出明显更强的鲁棒性，例如在有干扰物时能更好地识别正确物体并规划精确姿态 。

    - **以小胜大**：尽管参数量比 RT-2-X 小7倍（7B vs 55B），OpenVLA 在 BridgeData V2 的29个任务评估中，绝对成功率平均反而高出 **20.0%** ；在谷歌机器人评估中，OpenVLA 平均高出 **6.7%**。

    - **性能优势来源**：OpenVLA 的优越性能被归因于三个主要因素：更大规模的训练数据（970k vs 350k）、更仔细的数据清洗（如过滤无效动作）以及更优越的融合视觉编码器架构 。

- **研究问题二：OpenVLA 能否被有效微调至新任务，其与SOTA模仿学习方法的对比如何？**

  - **对比基线模型 (Baselines)**：
    - `Diffusion Policy`：一种 SOTA 级别的数据高效**模仿学习**方法，在目标任务的小数据集上**从零开始训练** 。
    - `Diffusion Policy (matched)`：一个输入输出格式与 OpenVLA 完全匹配的 Diffusion Policy 版本，用于更公平的比较 。
    - `Octo (fine-tuned)`：之前的 SOTA 通用模型，在同样的小数据集上进行微调 。
    - `OpenVLA (scratch)`：一个重要的**消融实验**版本，它使用与 OpenVLA 相同的基础 VLM，但**跳过了大规模机器人预训练**，直接在目标任务的小数据集上微调，用以衡量预训练的价值 。
    - `OpenVLA (fine-tuned)`：本文提出的方法，在小数据集上进行微调 。
  - **实验设置**：
    - 在一个全新的机器人平台（Franka 机械臂）上进行测试，该平台未出现在大规模预训练数据中 。
    - 使用小规模数据集（每个任务仅10-150个演示数据）进行全参数微调 。
    - 任务分为两类：**“窄泛化的单指令任务”**（如“把胡萝卜放进碗里”）和**“宽泛化的多指令任务”**（如根据指令在多个物体中选择一个进行操作）。
  - **实验结果与结论**：
    - **OpenVLA 综合性能最佳**：在所有任务的平均成功率上，OpenVLA 表现最好，证明了其作为下游任务默认学习策略的强大潜力 。
    - **术业有专攻**：从零训练的 Diffusion Policy 在窄泛化的单指令任务上表现出色，轨迹更平滑精确 。但在需要语言理解的、更复杂的多指令任务上，经过预训练的 OpenVLA 和 Octo 则表现更优 。
    - **预训练价值再次凸显**：`OpenVLA (scratch)` 的性能远低于经过预训练的 OpenVLA，这清晰地证明了大规模机器人预训练对于提升模型适应新任务的能力至关重要 。
    - **鲁棒性强**：OpenVLA 是唯一一个在所有测试任务上成功率都超过50%的模型，表明其在不同类型的任务间具有很强的稳定性。

* **研究问题三：能否通过参数高效微调和量化技术降低OpenVLA的计算需求？**

  - **对比方法**：
    - **参数高效微调**：比较了全参数微调、只微调最后一层、冻结视觉编码器、三明治微调和 LoRA微调 。
    - **量化**：比较了默认的 **bfloat16** 半精度、**8-bit** 整数 和 **4-bit** 整数三种推理精度 。

  - **实验设置**：
    - 参数高效微调实验在部分 Franka-Tabletop 任务上进行，评估成功率、训练参数量和显存占用 。
    - 量化实验在部分 BridgeData V2 任务上进行，评估成功率、显存占用和在不同消费级/服务器级 GPU 上的推理速度（动作/秒）。

  - **实验结果与结论**：
    - **LoRA 微调效果显著**：LoRA 实现了最佳的性能-计算权衡 。它以**仅训练1.4%参数**的极低成本，达到了与全参数微调**几乎相同的性能** 。这使得在单块 A100 GPU 上微调 OpenVLA 成为可能，计算量减少了8倍 。
    - **4-bit 量化兼顾性能与效率**：4-bit 量化可以在**不牺牲任务成功率**的情况下，将推理所需的显存占用**减少一半以上**（从约15-17GB降至7GB）。这使得在单块消费级GPU（如RTX 4090）上高效运行 OpenVLA 成为现实 。
    - **8-bit 量化陷阱**：8-bit 量化虽然减少了显存，但由于其操作的开销，反而**降低了推理速度**，并因此导致了任务成功率的显著下降。

* **局限性与未来工作**：

  - **输入模态单一**：目前 OpenVLA 只支持**单张图像**作为观测输入，无法处理多张图像输入，无法利用机器人自身的本体感知信息（如关节角度、力矩传感器读数），更没有考虑历史观测（无法形成对动态过程的记忆）。

  - **推理速度不足**：OpenVLA 模型的“反应速度”还不够快，难以满足像 ALOHA 系统那样要求高频率 (50Hz) 控制的场景。当前的推理速度限制了其在需要快速反应的灵巧任务（尤其是双手协同任务）上的应用。未来可探索采用动作分块（一次性预测未来一小段的多个动作）或推测解码等技术来加速模型的推理过程，让机器人反应更快。
  - **可靠性有待提升**：虽然 OpenVLA 已经是SOTA，但在测试任务中，其成功率通常低于90%，尚未达到非常高的可靠性。
